Hectic Hospital — 6-Hour Sprint Plan

3-Person Team Breakdown · Babylon.js + Havok Physics

Important: Engine Note

This repo uses Babylon.js, not Three.js. The architecture supports everything you need — just be aware so nobody wastes time trying to swap engines.

What We're Building

A top-down hospital pipeline sim. Patients enter, get checked in, diagnosed, and treated. You earn money on heals, lose money on lawsuits. Chaos ensues — bumping, knockbacks, accidental injuries. Think:

Two Point Hospital + Overcooked chaos + "Stabby McStab" surprises
Patient Pipeline
ENTER → RECEPTION → WAITING → ROOM → TREATMENT → EXIT

Hour-by-Hour

HourTask
0–1Rewrite GameModel (money, patients array, staff array). Define patient/staff interfaces. Stub out PatientPipeline state machine. Delete sample game code from PlayState.
1–2Build NPC agent loop: each agent has a role, state, target, update(dt). Receptionist accepts/rejects. Nurse picks up patient. Doctor treats. All driven by simple state machines.
2–3Wire agents to P2's waypoints (room positions). NPCs move between waypoints based on pipeline state. Patient spawner on a timer.
3–4Player interaction: tap NPC → show info panel. Tap staff → redirect them. Override receptionist. Basic money: +$$ on heal, -$$ on lawsuit.
4–5Polish the pipeline. Add "dangerous patient" archetype (rare). Connect P3's collision events → injury. Results state when shift ends.
5–6Integration, bug fixes, demo prep.

Hour-by-Hour

HourTask
0–1Start with circle-collider kinematic (the repo already does this). Each NPC gets a position, radius, velocity. Circle-circle resolution for all characters. Skip Havok for characters.
1–2Implement mass-based push: when two circles overlap, apply separation force weighted by mass. Fat character → high mass → pushes others. Fast character → high velocity → bigger impact. Add onCollision callback.
2–3"Drag patient" mechanic: nurse attaches to patient, patient follows with slight lag (spring/lerp). If nurse collides hard while dragging → patient gets injured (emit event).
3–4Animation controller: define states (idle, walk, interact, stunned/fall). Hook into P2's character meshes. Play correct animation based on agent state.
4–5Chaos tuning: knockback threshold, stun duration, injury severity. Make it feel funny, not realistic.
5–6Integration, physics bugs, animation polish.
Contract with P1:
  • Export updatePhysics(entities[], dt) that P1 calls each frame
  • Export onCollision: (a, b, force) => void callback
  • Export startDrag(nurse, patient) / stopDrag(nurse)
  • Export setAnimState(entity, state)

MVP Scope — What's Realistic in 6 Hours

Achievable (build this)

  • 4 visible rooms with basic props
  • Patients auto-enter, flow through pipeline
  • Receptionist auto-accepts (player can reject)
  • Nurse moves patient to room (with drag physics)
  • Doctor treats (success/fail dice roll)
  • Money counter goes up / down
  • NPCs bump into each other → knockback
  • 1 "dangerous patient" type (Stabby McStab)
  • Tap to select NPC + see info
