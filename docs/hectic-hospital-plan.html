Hectic Hospital — 6-Hour Sprint Plan

3-Person Team Breakdown · Babylon.js + Havok Physics

Important: Engine Note

This repo uses Babylon.js, not Three.js. The architecture supports everything you need — just be aware so nobody wastes time trying to swap engines.

What We're Building

A top-down hospital pipeline sim. Patients enter, get checked in, diagnosed, and treated. You earn money on heals, lose money on lawsuits. Chaos ensues — bumping, knockbacks, accidental injuries. Think:

Two Point Hospital + Overcooked chaos + "Stabby McStab" surprises
Patient Pipeline
ENTER → RECEPTION → WAITING → ROOM → TREATMENT → EXIT

Hour-by-Hour

HourTask
0–1Rewrite GameModel (money, patients array, staff array). Define patient/staff interfaces in types.ts. Stub out PatientPipeline state machine. Delete sample game code from PlayState.
1–2Build NPC agent loop: each agent has a role, state, target, update(dt). Receptionist accepts/rejects. Nurse picks up patient. Doctor treats. All driven by simple state machines.
2–3Wire agents to P2's waypoints. Patient spawner. Kinematic nurse-grab system: tap nurse → control movement → auto-attach nearby patient → tether follow. (You handle this basic physics, P3 upgrades it later with ragdoll attach.)

Hour-by-Hour

HourTask
0–1Circle colliders + mass-based push. Each NPC gets position, radius, velocity, mass. Circle-circle resolution. onCollision callback for P1. Heavier characters push lighter ones.
1–2Ragdoll prototype. When a character gets hit hard (velocity × mass > threshold), transition from animated state to ragdoll. Use Havok physics bodies on a simple bone chain (torso + limbs). Coordinate with P2: get character bone structure / skeleton info from P2's models.
2–33D grab / attach system. When nurse grabs patient: create a physics constraint (distance joint or ball-and-socket) between nurse and patient. Patient should swing/dangle realistically. Dragging a ragdolled patient = comedy gold. This replaces P1's simple lerp tether.
3–4Animation state machine + ragdoll blend. States: idle, walk, stunned, ragdoll, grabbed. Blend from animated → ragdoll on knockback. Blend back from ragdoll → standing on recovery. Hook into P2's character meshes/skeletons.
4–5Chaos tuning + injury system. Knockback threshold, stun duration, ragdoll recovery time. Collision force → injury severity → emit HospitalEvent to P1. Bumping while dragging a patient = extra chaos.
5–6Integration with P1's pipeline. Replace P1's kinematic tether with your grab system. Bug fixes, physics tuning.
Contract with P1 & P2:
  • Export updatePhysics(entities[], dt) that P1 calls each frame
  • Export onCollision: (a, b, force) => void callback → P1 uses this for injuries + pipeline
  • Export grabAttach(nurse, patient) / grabRelease(nurse) → replaces P1's simple tether
  • Export triggerRagdoll(entityId) / recoverFromRagdoll(entityId)
  • From P2: Need character skeleton/bone info from P2's models to attach ragdoll bodies. Coordinate at hour 1–2.

MVP Scope — What's Realistic in 6 Hours

Achievable (build this)

  • 4 visible rooms with basic props
  • Patients auto-enter, flow through pipeline
  • Receptionist auto-accepts (player can reject)
  • Nurse grab mechanic: tap nurse → control → auto-attach patient → drag to room
  • Ragdoll on hard collision (P3's signature mechanic)
  • 3D grab/attach: nurse physically holds patient with constraints, patient dangles
  • Doctor treats (success/fail dice roll)
  • Money counter goes up / down
  • NPCs bump into each other → mass-based knockback
  • 1 "dangerous patient" type (Stabby McStab)
  • Patience decay: patients leave angry if they wait too long
