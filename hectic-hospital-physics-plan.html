Physics & Ragdoll Plan

P3 Deep-Dive · Hectic Hospital · Babylon.js + Havok

Your Mission

Build the ragdoll grab system — the signature mechanic that makes the game hilarious. Nurses physically grab patients, drag them around, bump into things. Patients go ragdoll when hit hard. This is what makes clips shareable.

Ragdoll chaos + 3D grab attach = comedy gold
What P1 already built (your starting point)

P1 has a basic kinematic nurse-grab system working in PlayState.ts:

  • Tap a nurse → player controls her movement
  • Auto-attach patient when nurse walks near → patient follows with lerp tether
  • Drop patient in a room → fast-tracks to treatment

This is a placeholder. Your job is to build a physics-based version that replaces it at integration (hour 4–5). Same API contract, much better feel.

Coordination with P2 (Graphics)

Your ragdoll system needs bone/skeleton data from P2's character models. Talk to P2 in the first 15 minutes:

  • At minimum you need: torso, head, upper arm L/R, forearm L/R, upper leg L/R, lower leg L/R
  • If P2's models don't have skeletons yet, start with box-limb ragdolls (boxes for each body part, physics joints connecting them) — you can skin them to P2's meshes later
  • P2 can export skeleton info via mesh.skeleton.bones in Babylon.js

1 Character Physics CharacterPhysics.ts

Circle-circle collision + mass-based separation forces. This is the foundation everything else builds on.

What It Does

FeatureDetail
Circle collidersEach NPC has position, radius (Tuning.npcColliderRadius), mass, velocity
Overlap resolutionWhen two circles overlap, push them apart weighted by mass. Heavier = harder to move.
Knockback forceOn collision: force = velocity.length() * mass. If above Tuning.knockbackThreshold → trigger ragdoll on the lighter entity.
Stun stateAfter knockback, entity is stunned for Tuning.stunDurationSec. Can't move.

API Contract

export class CharacterPhysics { addEntity(id: string, root: TransformNode, radius: number, mass: number): void; removeEntity(id: string): void; update(dt: number): void; // P1 calls this every frame onCollision(cb: CollisionCallback): void; // P1 subscribes for injury events getEntity(id: string): PhysicsEntity | undefined; dispose(): void; }
Implementation tip

Don't use Havok for this layer. Keep it kinematic (manual position updates). Havok is only for ragdoll bodies in System 2. This avoids fighting two physics systems.

2 Ragdoll System RagdollSystem.ts needs P2 skeleton data

When a character gets hit hard, they transition from animated mesh to a physics-driven ragdoll. This is the money shot.

Two Approaches (pick based on P2's models)

ApproachWhen to useComplexity
A: Box-limb ragdoll P2's models have no skeleton, or you want to start fast Simpler, faster to build
B: Skeleton-mapped ragdoll P2's models have bones/skeleton (e.g. Kenney animated characters) Better looking, more complex

Approach A: Box-Limb Ragdoll (recommended for hackathon)

Create physics bodies (boxes) for each limb, connect with Havok distance/ball-and-socket joints. On ragdoll trigger, hide the animated mesh, show the ragdoll bodies, copy the pose.

// Ragdoll bone chain (Havok physics bodies) // // [head] ← sphere, small mass // | // [L arm]─[torso]─[R arm] ← boxes, ball-and-socket joints // | // [L leg] [R leg] ← boxes, hinge joints (knees) // // Total: 7 bodies, 6 joints // All created once per character, hidden until ragdoll triggers

Ragdoll Lifecycle

PhaseWhat happens
NormalAnimated mesh visible, ragdoll bodies hidden + kinematic (follow bone positions)
TriggerHide animated mesh. Enable ragdoll bodies as dynamic. Apply impact force. state = "ragdoll"
Ragdoll activeHavok simulates. Bodies tumble. Timer counts down.
RecoveryFreeze ragdoll. Lerp animated mesh back to standing pose. Re-enable kinematic. state = "idle"

API Contract

export class RagdollSystem { createRagdoll(entityId: string, root: TransformNode, scene: Scene): void; triggerRagdoll(entityId: string, impactForce: Vector3): void; recoverFromRagdoll(entityId: string): void; isRagdolling(entityId: string): boolean; update(dt: number): void; // auto-recover after timer dispose(): void; }

3 3D Grab / Attach System GrabSystem.ts replaces P1's tether

The signature mechanic. Nurse grabs a patient — physically. Patient's body trails behind, bumps into things, ragdolls if hit hard while being dragged. This is what people will clip.

How It Works

PhaseWhat happens
GrabP1 calls grabAttach(nurseId, patientId). Create a Havok distance constraint between nurse root and patient root. Set max distance = Tuning.nurseTetherOffset.
DraggingNurse moves (kinematic, P1 controls). Patient body is pulled by constraint. Patient can swing side to side, bump into walls/NPCs.
Bump while draggingPatient hits another NPC → collision → potential ragdoll chain reaction on the hit NPC. Comedy.
ReleaseP1 calls grabRelease(nurseId). Remove constraint. Patient stays where dropped.
Fallback: spring-follow (if Havok constraints are too finicky)

If Havok distance joints don't work well: use a spring-follow system instead. Each frame, compute a force pulling patient toward nurse. Apply it as velocity. This is simpler and still looks physical enough.

API Contract

export class GrabSystem { grabAttach(nurseId: string, patientId: string): void; grabRelease(nurseId: string): void; isGrabbing(nurseId: string): boolean; getGrabbedPatient(nurseId: string): string | null; update(dt: number): void; dispose(): void; }

4 Animation State Machine CharacterAnimator.ts needs P2 anims

Controls which animation plays based on agent state. Handles the tricky animated → ragdoll → animated transitions.

States

IDLE ↔ WALK → STUNNED → RAGDOLL → RECOVERING → IDLE
IDLE → GRABBED (dragged by nurse)

Transition Logic

FromToTrigger
idle / walkstunnedCollision force > threshold but < ragdoll threshold
anyragdollCollision force > ragdoll threshold (from CharacterPhysics)
ragdollrecoveringRagdoll timer expires
recoveringidleRecovery animation finishes (or just a short timer)
anygrabbedGrabSystem attaches this entity
grabbedidleGrabSystem releases this entity
If P2's models have no animations

Fake it with transforms: idle = slight bob (y += sin(time) * 0.05), walk = faster bob + lean forward, stunned = tilt sideways + flash color, ragdoll = physics takes over. This still feels good.
